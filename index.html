RSGAN论文笔记

Posted on 2020-06-30 Edited on 2020-07-07

Introduction

研究发现，基于社交信息的推荐系统有时候会降低推荐系统的性能。 原因在于社交关系的多样性，和不稳定性。为了解决这个问题，一些研究将注意力转向从explicit社交网络中提取有益的社交关系。然而社交关系几乎和 user-item matrix 一样稀疏，过滤后提取的这些社交关系不足以提高推荐性能。基于这个原因，一个更好更灵活的方法是寻找 implicit friends（拓扑结构不相连，但有相似的偏好）。但是没有一个研究能够衡量这些implicit friends的质量。此外，几乎所有的社交推荐算法都将traing建模成一个静态的过程，忽略了训练过程中两个user的相似的的变化。由于参数和模型的复杂性，在经过多次优化迭代后，一些关系不可避免地不利于模型，甚至成为噪声。因此，作者认为应该开发技能发现reliable friends，又能动态的评估朋友间关系的方法。

作者提出了一个专注于发现reliable friends，并且动态评估这些friends质量的推荐系统框架。首先作者根据【17】中提到的识别implicit friends的方法，找到这些用户作为 seeded friends （【17】的内容可以看我IF-BPR论文笔记） 。然后作者提出了一个基于生成式对抗网络（GAN）的框架，该方法能够在种子好友的监督下动态的识别可靠的社会关系，并提高推荐质量。

THE PROPOSED FRAMEWORK

Preliminaries

$R^{m\times n}$ : user-item rating matrix

损失函数：
$L_{BPR}=- \sum_{(u,j,j))}^{}log\sigma (x_{ui}(\Phi )-(x_{uj}(\Phi )+\lambda_{\Phi }\left | \Phi \right |^{2}))$

Motivation

之前的的大部分社交推荐算法都认为社交的影响是静态的。 但作者认为在训练过程中，应当对社交关系进行实时的评估。作者用 off-line decision making 和 模型训练做了个对比。user会询问朋友并得到不同的意见，一开始他会同意一些人的意见，但由于复杂的心理活动，最后会倾向和另一群人达成共识。同样的，在训练过程中，社交推荐系统也是在一直进化的。一些社交关系最初可能是有益的，但随后可能对模型有害。作者把模型参数更新的过程类比成人类的心里活动。（好像是有那么点道理，深度学习这么智能的么TOT…）

Model Overview

在本节中，作者将介绍基于生成式对抗网络的可靠社交推荐框架（RSGAN）。

模型有一个基本的假设，就是users对他们的reliable friends消费的items感兴趣。为了找到这些items,首先需要为用户找到reliable friends，这也是RSGAN要解决的关键问题。 如图1所示，RSGAN由两部分组成：generator（$G_{\theta}$）和 discriminator ($D_{\Phi}$)。discriminator负责ranking候选的item，为用户生成推荐列表。generator负责生成可靠的friend，然后sample一个被该friend消费的item，去增强discriminator。同时，如果generator生成的friend所消费的item对增强discriminator没有帮助的话，discriminator会惩罚这个friend，并向generator返回梯度以降低产生该朋友的概率。

Friend and Item Generation

为了指导generator生成质量更好的friend，作者首先获取一些seeded friends作为superviosr(ground truth)。获取seeded friends的方法，作者利用了[17]中perfect friends的获取方法。（【17】的内容在IF-BPR论文笔记中有详细介绍） 但是，seeded friends的数量非常有限，因此，第一个挑战是如何根据seeded friends来重建social profile，以便有更大范围的朋友被选择。作者选择CDAE[44]作为generator。

在收集到每个用户的seeded friends之后，作者将他们编码成binary vecotrs，作为不完整的用户social profile。这些不完整的用户配置文件被作为初始输入输入到生成器中。CDAE以种子好友为正例，缺失好友为负例，利用交叉熵损失对内部神经元进行优化，重建可靠social profile，然后softmax层输出整个用户集U的概率分布。

$P_{u}(v|u,S_{u})=\frac{exp(c(v|S_{u}))}{\sum exp(c(v^{‘}|S_{u}))}$

在已知friends的概率分布的情况下，sample概率最大的用户作为user的reliable friend。但是，有两个问题。首先，同一个user在再一次被选择的概率很大，导致overfitting。其次，离散采样过程是非差分的，这意味着训练不是端到端，导致较差的推荐性能。为了解决这个问题，作者将两层Gumbel-softmax集成到框架中，模拟friend和item的采样过程。Gumbel-softmax这里的解释没太看懂，所以贴了作者的源代码：

CONCLUSION

社交推荐算法的问题是社交信息稀疏并且不可靠。作者提出了基于GAN模型的可靠朋友生成架构（RSGAN）。通过generator和discrimaniator之间的竞争，RSGAN可以动态地惩罚不可靠的社会关系，自适应地产生可靠的朋友，从而更好的预测用户的偏好。为了实现端到端训练，作者采用Gumbel-Softmax来放松离散采样。通过实验，证明了RSGAN的优越性，并验证了所生成的可靠隐式朋友的积极作用。

IF-BPR论文笔记

Posted on 2020-06-18 Edited on 2020-06-23

Introduction

在目前社交推荐算法中，大部分都基于一个假设：相连的用户偏好相似，而不相连的用户则偏好不同。但现实世界中，社交网络上相距很远的用户也可能具有相似的偏好，这样的用户被称为implicit friends。作者提出了一个新的基于heterogeneous information network（HIN）的社交推荐方法，为每个用户识别出有最大相似度的implicit friends，通过这些implicit friends来enhance the item recommendation.

PRELIMINARES

Heterogeneous Information Network

$G_{s}$: user-user social network

$G_{r}$: user-item bipartite network

作者将用户的$G_{s}$和$G_{r}$拼接起来，整体作为一个Heterogeneous Information Network。Fig.1 is an illustration of HIN. HIN里面包含user和item两种节点，三种关系包括user-item和user-user关系。

本文与其他work不同的点在于，作者利用了negative feedbacks。其中引用[8]介绍了一种生成negative sample的方法：This is done by selecting a negativity threshold value, such that the values of ratings above this threshold are treated as positive examples and all other values - as negative. 后面实验部分有具体介绍本文的negative sample strategy。

Implicit Friends

Implicit Friends 是那些有相似偏好，但在网络结构上没有直接相连的user。

Paper的目的是找到每个user的implicit friends，利用这些implicit friends 去提高算法的性能。

IDENTIFYING THE IMPLICIT FRIENDS OVER HIN

算法首先生成基于随机游走的biased meta-path，然后通过heterogeneous skip-gram来学习节点的embedding representations，为每个user识别Top-K个implicit friends。

Generating Social Corpora over HIN

由于HIN is in large-scale, 所以第一个挑战是降低计算成本的同时保存原始网络的信息。
作者定义了HIN上的六种meta-paths，然后基于随机游走的biased meta-paths生成节点序列来解决问题。

然而，social relations往往是有噪声的，因此我们必须寻找具有biased probability的可靠序列。下面作者展示了如何利用元路径来引导随机游走产生有biased的节点序列，即social corpus。

这里作者给出的解释是，在随机游走的每一步，下一个节点的类型由预先定义的meta-paths决定。如果前后节点分别是U和I类型(或者相反的顺序)，则选择后续节点。但如果前后节点都是U类型，则根据邻居节点的重叠程度来决定后续节点。也就是后续节点和前一个节点应当有大量相同的朋友。这样就可以获得更可靠的序列，减少嘈杂的社会关系的不良后果。

基于随机游动的网络表示学习[9]通常根据节点在上下文窗口中的共现情况来学习节点的表示。

(这里我研究了下源代码，positive feedbacks和negative feedbacks都是根据上文引用[8]里面介绍的生成negative sample的方法生成将rating数据分成positive feedbacks和negative feedbacks两部分。)

user的preference有positive feedbacks和negative feedbacks。在进行随机游走时，作者利用positive feedbacks和用户关系来生成positive social corpus。同样，利用negative feedbacks和用户关系生成negative social corpus负面社会语料库是建立在负面反馈和用户关系上的。这两个social corpus中隐含了用户的全部偏好(positive preference 和 negative preference)，最后用于生成user不同的表示。

Learning Node Representations and Identifying Top-K Implicit Friends

利用上面的到的node sequence，就可以利用heterogeneous Skip-Gram[5]模型，学习node embedding。（和应用word2vec模型相似，将social corpus看成语料库，每个节点都是单词，node sequence相当于一句话，学习表示。） 知道node embedding就可以计算余弦相似度了。因为有positive和negative两种social corpus，所以可以找到每个用户的Top-K个positive和negative friends。positive friends就是与当前user具有相似的positive preference的implict friends，而negative friends就是与当前user具有相似的negative preference的用户。

IF-BPR:BPR WITH IMPLICIT FRIENDS

Adaptive Refinement for Top-K Implicit Friends

作者提出了IF-BPR模型，该模型将implicit friends加入到BPR中，并根据不同类型的朋友对item进行排序。

在识别出每个user的Top-K个positive和negative的implicit friends后，它们的部分是重叠的，这部分重叠的friends就是perfect friends。他们与当前user在positive和negative上的偏好都相似。因此，每个用户有三种implicit friends——implicit perfect friends, implicit positive friends, 和implicit negative friends。

为什么是Docker？

软件开发最大的难题是配置环境，不同的程序可能需要不同的配置环境。虚拟机是一种解决方案，用户可以通过不同的虚拟机实现不同的环境配置。但由于虚拟机占用资源高，Linux提供了另一种解决方案，Linux容器。Docker属于Linux容器的一种封装，提供简单易用的容器使用接口。Docker将应用程序与该程序的依赖，打包在一个文件里，运行这个文件，就会生成一个虚拟容器。程序在这个虚拟容器里运行，就像在真实的物理机上运行一样。

镜像（Image）& 容器（Containner）

Image是一个特殊的文件系统，它提供了容器运行时所需要的程序、库资源、配置等文件，还包含了为运行时准备的一些配置参数。Docker把这些文件打包在image里面，通过这个文件，生成Container，你的程序都在这个Container里运行。
